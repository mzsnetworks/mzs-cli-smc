## Thread option

1/ Every script you automate is a liability you now own. We only ever count the asset. 🧵

2/ Automating a task feels like pure win — did it once, now it runs itself. What nobody logs is the software you now have to maintain, understand, and trust for life. That's automation debt.

3/ The interest:
• the playbook that breaks on every vendor API change
• the script only one person understands
• the pipeline everyone's afraid to touch

4/ Manual work degrades gracefully — a human notices. Unmaintained automation fails silently, at scale, on stale assumptions. The blast radius is bigger than the toil it replaced.

5/ Automation you don't maintain isn't saving time. It's deferring a bill.

Automate less, but own what you automate.
