Every script you automate is a liability you now own. We only ever count the asset.

Automating a task feels like pure win — you did it once, now it runs itself. What nobody logs is what you took on: software that has to be maintained, understood, and trusted for life, or it becomes a hazard.

That's automation debt. It accrues interest — the playbook that breaks on every vendor API change, the script only one person understands, the pipeline everyone's afraid to touch.

Manual work degrades gracefully; a human notices when something's off. Unmaintained automation fails silently, at scale, on stale assumptions. The blast radius is bigger than the toil it replaced.

Automation you don't maintain isn't saving time — it's deferring a bill. Automate less, but own what you automate.
