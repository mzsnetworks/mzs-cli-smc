## Thread

1/ 5 signs your "NetDevOps transformation" is just Git with extra steps 🧵

2/ Configs in a repo — pushed by hand. Truth is stored in Git but decided in a CLI session. That's a backup folder with a commit history.

3/ PRs approved unread. 400 lines, "LGTM" in 40 seconds. Review became a turnstile, not a gate.

4/ A green pipeline nobody trusts. Checks pass — then someone SSHes in to verify anyway. It tests what was easy, not what fails.

5/ A branch named prod-final-v2. Branch names are process ambiguity written down.

6/ Rollback = SSH. Forward path automated; reverse path is a senior engineer typing fast at 2am. A one-way door at machine speed.

7/ Same root cause: the tool got adopted, the behavior didn't.

Git stores your configs. It can't store your discipline. Which sign stung?
