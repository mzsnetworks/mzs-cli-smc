# Facebook — git-with-extra-steps (MZS page)

Five signs your "NetDevOps transformation" is just Git with extra steps:

1️⃣ Configs in a repo — pushed by hand
2️⃣ PRs approved unread
3️⃣ A green pipeline nobody trusts
4️⃣ A branch named prod-final-v2
5️⃣ Rollback = SSH

Same root cause every time: the tool got adopted, the behavior didn't. Git arrived; the decisions stayed in terminals, heads, and habit.

Git stores your configs. It can't store your discipline. A transformation is measured by what changes at 2am — not by what moved into a repo.

Which sign stung? Be honest — ours was #3 for a while. 👇
