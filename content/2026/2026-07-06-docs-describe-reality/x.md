## Option B — thread

1/ The most dangerous document in your network isn't the one that's missing.

It's the one that's confidently wrong. 🧵

2/ Every diagram describes one moment: design day.

Production diverged the week it went live — the 11pm emergency change, the "temporary" VLAN from three years ago, the failover rewired mid-outage and never redrawn.

3/ Wrong docs are worse than no docs.

• No docs: the 2am troubleshooter verifies everything. Slow, but honest.
• Wrong docs: they trust — and walk the "documented" failover path while the real one burns.

4/ I've watched a migration go exactly to plan — then weeks of config inconsistencies surfaced, accumulated over years, none on any diagram.

Production never looks like the drawings.

5/ The fix isn't drawing discipline. Nobody updates Visio at 11pm. Everybody makes changes at 11pm.

Generate docs FROM the network. Reconcile source of truth against live state. A doc produced by the network is evidence; one produced about it is a memory.

6/ Intent lives in diagrams.
Decisions live in configs.
Truth lives only in the running network.

Document the truth.
