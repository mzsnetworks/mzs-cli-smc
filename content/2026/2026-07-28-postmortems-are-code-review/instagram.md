"Human error" is disappearing from postmortems. Not because mistakes stopped. ⬇️

Because the mistake is now a commit — with an author, a reviewer who approved it, tests that passed it, and a pipeline that shipped it.

Blame the human and you have to explain the other three.

Google's SRE book: roughly 70% of outages come from changes to a live system. The change used to be keystrokes nobody watched. Now it's a diff. So the postmortem stops interrogating a person and starts interrogating a change — which is exactly what code review does, in a different tense.

DORA has said it for years: failure should lead to inquiry, not scapegoating. Remove the blame, you remove the fear — and problems surface while they're still cheap.

🔁 Code review is the postmortem before the incident.
🔁 The postmortem is the code review after.
The best teams make them the same conversation.

When did "human error" last close one of your postmortems? 👇
