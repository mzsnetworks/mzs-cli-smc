## Thread option

1/ A runbook nobody runs isn't documentation. It's fiction with a table of contents. 🧵

2/ A runbook is only true the day it's written. The network keeps moving — the jump host in step 4 gets decommissioned, the CLI changes in a firmware upgrade — and the doc has no idea the world moved.

3/ You find out at 3am. Step 4 references a server gone since last year. Now you're debugging the outage and the runbook at once.

4/ Two ways to keep one honest:
• automate it, so it fails loudly when it drifts
• rehearse it on a quiet Tuesday

5/ The only runbook you can trust is one that gets exercised. Everything else is confidence you haven't earned.

Which "critical" runbook has nobody run in the last year?
