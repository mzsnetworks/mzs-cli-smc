A runbook nobody runs isn't documentation. It's fiction with a table of contents. 📖

We write the failover procedure, file it in the wiki, and feel safer. The safety is imaginary.

A runbook is only true the day it's written. The network keeps moving:
→ someone decommissions the jump host in step 4
→ the IP in step 7 gets re-used
→ the CLI syntax changes in a firmware upgrade

None of it updates the doc. It goes stale wearing the authority of something once correct.

You find out at 3am — step 4 points to a server gone since last year. Now you're debugging the outage AND the runbook ⚙️

The value was never in writing it. It's in running it.

Two ways to keep one honest:
→ automate it, so it fails loudly when it drifts
→ rehearse it on a quiet Tuesday

The only runbook you can trust is one that gets exercised.

Which "critical" runbook has nobody actually run in the last year? 👇
